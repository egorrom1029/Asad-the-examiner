* Стек разработки: Gemini + Replit.
* Технология интеграции ИИ: Использование Gemini Multimodal Live API через WebSocket-соединение для реализации нативного потокового обмена аудио (Audio-to-Audio) без промежуточного преобразования в текст.
* Логика ИИ: Вся бизнес-логика, характер профессора, правила ведения допроса и критерии оценки зашиваются непосредственно в System Prompt на сервере.
* Архитектура фронтенда: Single Page Application (SPA), собранное в одном файле index.html без внешних зависимостей и фреймворков. Навигация между состояниями экранов управляется локальным JavaScript-стейтом.
* Серверная архитектура (Бэкенд):
  - Язык программирования сервера: НУЖНО УТОЧНИТЬ (в обсуждении упоминался выбор между Python и Node.js/JavaScript, финальный выбор не сделан).
  - Механизм проксирования запросов: НУЖНО УТОЧНИТЬ (рассматривается прямая отправка пакетов с фронтенда с обязательным использованием VPN пользователем, либо проксирование трафика через зарубежный сервер Replit во избежание региональных блокировок Google API).
* Точки интеграции: Существующие во фронтенде функции генерации случайных данных (генераторы вердиктов, ошибок и симуляции речи профессора) спроектированы под прямую замену на обработку событий WebSocket-соединения с бэкендом.
