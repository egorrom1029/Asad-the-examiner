* Стек разработки: Gemini + Google AI Studio.
* Технология интеграции ИИ: Использование Gemini Multimodal Live API через WebSocket-соединение для реализации нативного потокового обмена аудио (Audio-to-Audio) без промежуточного преобразования в текст на экране.
* Модель ИИ: models/gemini-2.0-flash-exp.

* Логика ИИ: Вся бизнес-логика, характер профессора, правила ведения допроса и критерии оценки зашиваются непосредственно в System Prompt на сервере. В конце сессии ИИ обязуется выдать текстовый блок с тегом [FINAL_VERDICT], содержащий текстовый вердикт и пронумерованный список строго из 3–5 логических ошибок.

* Архитектура фронтенда: Single Page Application (SPA), собранное в одном файле index.html без внешних зависимостей и фреймворков. Навигация между тремя состояниями (Экран ввода темы -> Экран экзамена -> Экран результатов) управляется локальным JavaScript-стейтом. Захват звука осуществляется через navigator.mediaDevices.getUserMedia с нарезкой аудио-чанков MediaRecorder каждые 100 мс для стриминга в реальном времени.

* Серверная архитектура (Бэкенд):
  - Язык программирования сервера: Python (FastAPI + websockets) с использованием asyncio для параллельной обработки входящего и исходящего аудио-потоков.
  - Механизм проксирования запросов: Проксирование трафика через зарубежный сервер Google AI Studio. Фронтенд подключается к эндпоинту бэкенда @app.websocket("/ws/exam"), а бэкенд перенаправляет бинарные данные в Google API, используя серверный токен GEMINI_API_KEY, что полностью решает проблему блокировок без участия пользователя.
  
* Логика WebSocket-моста:
  - Сразу после установления соединения с Gemini Live API бэкенд отправляет JSON-пакет конфигурации (setup message), передавая systemInstruction и устанавливая responseModalities: ["AUDIO"] вместе с конфигурацией голоса (например, "Puck"). Только после этого шага ИИ начинает говорить первым.
  - Бэкенд осуществляет непрерывный транзит: бинарные аудио-данные от фронтенда пересылаются в Gemini, а бинарные ответы Gemini возвращаются на фронтенд.
  - Бэкенд сканирует текстовые сообщения (text parts) от Gemini. При обнаружении тега [FINAL_VERDICT] бэкенд дожидается окончания передачи текстового фидбека, пересылает его на фронтенд и закрывает WebSocket-сессию.

* Точки интеграции: Существующие во фронтенде функции генерации случайных данных полностью заменены на обработку реальных событий WebSocket-соединения с бэкендом.
